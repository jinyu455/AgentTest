# 情绪分析 Agent 需求规格说明书（基于两轮确认结果）

**文档版本：** V1.0  
**编制日期：** 2026-04-22  
**依据：** 用户两轮需求回答，无额外脑补  
**适用范围：** 文本情绪分析智能体（Agent）全流程开发、测试、验收

## 1. 项目概述

### 1.1 项目目标

开发一个云端部署、后台自动调用的文本情绪分析 Agent，实现对用户聊天、评论、客服对话文本的基础情绪识别、反讽识别、混合 / 模糊情绪识别，输出结构化分析结果，支持运营、质检、审核人员使用，并提供统计监控与数据管理能力。

### 1.2 核心范围

- 仅支持文本输入情绪分析，不处理语音、图片、视频
- 支持 6 类基础情绪，后续可扩展
- 必须支持反讽、混合情绪、模糊压抑情绪识别
- 结果结构化 JSON 输出，自动同步第三方系统，数据保存 ≥ 6 个月
- 提供权限控制、调用日志、统计报表、异常监控

## 2. 角色定义

| 角色 | 职责 |
|---|---|
| 系统调用方 | 自动提交文本、接收分析结果、触发实时分析 |
| 普通运营 / 客服质检 / 内容审核 | 查看权限内分析记录、导出权限内数据、查看统计报表 |
| 管理员 | 配置数据权限、查看全量数据、全量导出、删除记录、接收告警 |

## 3. 功能需求（FR）

### FR-01 文本路由分类

**功能描述：** 对输入文本自动判断类型：直接表达 / 疑似反讽 / 混合复杂情绪，决定后续调用链路  
**参与角色：** 系统调用方、Agent 自动执行  
**前置条件：** 接收到有效文本  
**输入：** 用户文本、用户 ID、消息 ID、来源、时间

**处理流程：**
1. 接收文本
2. 判断是否直接情绪 / 疑似反讽 / 混合情绪
3. 输出路由结果与是否需要反讽 / 混合检查

**输出：** 路由类型、是否需要反讽检查、是否需要混合检查、路由原因  
**异常情况：** 空文本 / 乱码 / 超长文本→标记无效文本，路由为直接表达，情绪默认中性

### FR-02 表层情绪识别

**功能描述：** 对文本做表层情绪识别，输出情绪、强度、置信度、结构化词汇特征  
**参与角色：** Agent 自动执行  
**前置条件：** Router Agent 已完成分类  
**输入：** 原始文本、路由结果

**处理流程：**
1. 分词并提取情绪词、程度词、否定词、转折词
2. 判断主情绪
3. 给出强度 0–100、置信度 0–1
4. 输出分析原因

**输出：** tokens、emotion_words、emotion、intensity、confidence、reason  
**异常情况：** 无情绪文本→输出中性，强度 0–20

### FR-03 反讽识别与修正

**功能描述：** 对疑似反讽文本判断是否反讽，输出真实情绪并修正强度  
**参与角色：** Agent 自动执行  
**前置条件：** Router 标记 `need_sarcasm_check=true`  
**输入：** 原始文本、表层情绪结果

**处理流程：**
1. 判断是否反讽
2. 提取表层情绪与真实情绪
3. 修正强度与置信度

**输出：** is_sarcasm、surface_emotion、true_emotion、revised_intensity、confidence、reason  
**异常情况：** 无法判断反讽→按非反讽处理，沿用表层结果

### FR-04 混合 / 模糊情绪识别

**功能描述：** 对混合 / 复杂情绪识别主次情绪、混合比例、修正强度  
**参与角色：** Agent 自动执行  
**前置条件：** Router 标记 `need_mix_check=true`  
**输入：** 原始文本、表层情绪结果

**处理流程：**
1. 判断是否混合情绪
2. 提取主情绪、次情绪、比例
3. 修正强度与置信度

**输出：** is_mixed、primary_emotion、secondary_emotion、mix_ratio、revised_intensity、confidence、reason  
**异常情况：** 无明显混合→按单情绪输出

### FR-05 结果融合与最终输出

**功能描述：** 整合所有 Agent 结果，输出统一结构化 JSON  
**参与角色：** Agent 自动执行  
**前置条件：** Emotion/Sarcasm/Mix 执行完成  
**输入：** Router、Emotion、Sarcasm、Mix 结果

**处理流程：**
1. 按类型（direct/sarcasm_suspected/mix）决策最终情绪
2. 合并强度、置信度、原因
3. 生成统一结果

**输出：** 标准 JSON（含最终情绪、次情绪、强度、置信度、是否反讽、是否混合、原因等）  
**异常情况：** 部分 Agent 失败→优先可用结果，降低置信度

### FR-06 结果同步第三方系统

**功能描述：** 分析成功后自动同步到用户行为平台、客服管理系统  
**参与角色：** 系统自动执行  
**前置条件：** 分析完成且结果有效  
**输入：** 最终分析结果

**处理流程：**
1. 发起同步请求
2. 失败则重试最多 3 次，间隔 1 秒
3. 仍失败标记同步失败

**输出：** 同步成功 / 失败状态  
**异常情况：** 同步失败→记录日志，不阻塞主流程

### FR-07 数据存储与管理

**功能描述：** 原始文本与分析结果分表入库，保存 ≥ 6 个月  
**参与角色：** 系统自动执行、管理员  
**前置条件：** 分析完成  
**输入：** 原始文本、分析结果、调用信息

**处理流程：**
1. 写入原始文本表
2. 写入分析结果表
3. 支持按条件查询、删除、导出

**输出：** 入库成功 / 失败、记录列表、导出文件  
**异常情况：** 入库失败→记录错误日志，不丢失数据

### FR-08 调用日志记录

**功能描述：** 记录每一次分析调用，便于审计与排查  
**参与角色：** 系统自动执行  
**前置条件：** 收到调用请求  
**输入：** 调用时间、文本、结果、调用方、状态

**处理流程：** 自动落库保存  
**输出：** 日志列表  
**异常情况：** 日志写入失败→不影响主分析流程

### FR-09 权限控制

**功能描述：** 按角色与数据范围隔离数据，控制查看 / 导出权限  
**参与角色：** 管理员、普通运营 / 质检 / 审核  
**前置条件：** 用户已登录  
**输入：** 用户角色、数据权限配置

**处理流程：**
1. 校验用户角色
2. 按客服组 / 业务线 / 渠道过滤数据
3. 控制导出范围

**输出：** 权限内数据、无权限提示  
**异常情况：** 越权访问→拒绝并记录日志

### FR-10 统计报表与监控

**功能描述：** 提供情绪分布、调用量、准确率、超时率等统计与告警  
**参与角色：** 管理员、普通运营 / 质检 / 审核  
**前置条件：** 有分析数据与调用日志  
**输入：** 历史数据、统计周期、筛选维度

**处理流程：**
1. 按维度 / 周期统计
2. 实时数据 5 秒刷新，小时 / 天级每小时更新
3. 达到阈值触发告警

**输出：** 统计图表、报表、导出 Excel/CSV、告警信息  
**异常情况：** 统计数据异常→标记异常，不中断服务

## 4. 业务规则（BR）

### BR-01 情绪标签规则

初始支持：开心、悲伤、愤怒、焦虑、厌烦、中性；后续可扩展疲惫、失落、无奈。

### BR-02 反讽与混合情绪规则

- 正向词 + 负向事件、夸张赞美 + 抱怨语境→判定疑似反讽
- 转折、双情绪、模糊表达→判定混合情绪

### BR-03 文本去重规则

5 分钟内相同文本（MD5 一致）只分析一次，返回缓存结果，不重复入库、不计入统计。

### BR-04 超时规则

单条分析 ≥ 500ms 判定超时，返回中性、强度 0、置信度 0、原因“分析超时”，不自动重试。

### BR-05 无效文本规则

空文本、纯符号、乱码、超长文本→标记无效，情绪中性、强度 0、置信度 0。

### BR-06 数据存储规则

分析结果与原始文本入库，保存至少 6 个月。

### BR-07 同步重试规则

第三方系统同步失败，最多重试 3 次，间隔 1 秒，仍失败标记同步失败。

### BR-08 告警规则

- 1 分钟内连续超时 ≥ 20 次→告警
- 整体准确率 ＜ 80%→告警
- 超时率 ＞ 5%→告警
- 并发 ＞ 100 TPS→告警

## 5. 非功能需求（NFR）

### NFR-01 响应时间

单条分析响应时间 **≤500ms**。

### NFR-02 识别准确率

整体识别准确率 **≥85%**；反讽 / 混合情绪准确率 **≥80%**。

### NFR-03 并发能力

峰值支持 100 TPS（每秒 100 次调用）。

### NFR-04 部署方式

云端部署，不支持本地 / 私有化。

### NFR-05 架构要求

轻量架构，避免无意义重复调用，控制成本。

### NFR-06 数据安全

按角色权限隔离数据，操作可审计，不修改分析结果。

## 7. 约束与假设

1. 所有输入为合法 UTF-8 文本，不处理特殊编码
2. 敏感内容由外部系统处理，本 Agent 不做过滤
3. 分析规则全场景统一，暂不支持场景化配置
4. 调用方负责文本合法性，本 Agent 不做内容安全审核
